# Лабораторная работа 4: стохастическая оптимизация

Индивидуальный исследовательский трек команды: **Трек 2. SAGA: сокращение дисперсии без внешних циклов**.

В работе реализованы и исследованы стохастические методы оптимизации для ML-задач с L2-регуляризацией:

- Mini-batch SGD с расписаниями шага `constant`, `inverse_sqrt`, `step_decay`;
- SVRG;
- Adam;
- SAGA.

Командный пакет оракулов: **№1 — Log-Cosh + Exponential**.

## Структура проекта

```text
.
├── figs/                 # сохраненные графики экспериментов
├── notebooks/            # отдельные ноутбуки по каждому эксперименту
├── reports/              # итоговый отчет
├── src/                  # код оракулов, оптимизаторов и утилит
```

Основные файлы:

- `reports/lab4_report.md` — итоговый отчет с формулами, графиками и выводами.
- `notebooks/01_sgd_schedules.ipynb` — эксперимент 1: расписания шага SGD.
- `notebooks/02_variance_reduction.ipynb` — эксперимент 2: SVRG и линейная сходимость.
- `notebooks/03_adam_scaling_and_anomalies.ipynb` — эксперимент 3: Adam, масштаб признаков и аномалии.
- `notebooks/04_track2_saga.ipynb` — исследовательский трек 2: SAGA.
- `src/oracles.py` — реализации Log-Cosh и Exponential оракулов.
- `src/optimization.py` — реализации GD, SGD, SVRG, Adam, SAGA.
- `src/experiments.py` — генерация данных, построение графиков, поиск референсного минимума.

## Запуск

Установить зависимости:

```bash
pip install -r requirements.txt
```

Открыть и последовательно выполнить ноутбуки из папки `notebooks/`. Каждый ноутбук сохраняет свои графики в папку `figs/`.

Проверить синтаксис модулей:

```bash
python -m compileall src oracles.py optimization.py
```

## Примечания к экспериментам

Ось X на графиках — число эффективных эпох. Один полный градиент стоит одну эпоху, а один mini-batch размера `b` стоит `b / m` эпохи.

Для SAGA таблица индивидуальных градиентов имеет размер `m x n`, поэтому в исследовательском треке используется не самый большой датасет. Среднее таблицы пересчитывается рекуррентно, без полного суммирования всех строк на каждой итерации.
